## 3. Hallazgos del audit que **NO** eran reales

Anotados aquí para que la próxima sesión no los persiga. Los auditores LLM no abrieron el código en estos cuatro:

### 3.1 Refresh token "podría no estar hasheado"

- **Claim**: "verificar que se almacena hasheado, si no es brecha".
- **Realidad**: [internal/auth/service.go:354](../../internal/auth/service.go) crea sesión con `RefreshTokenHash: hashToken(refreshToken)`. [internal/auth/service.go:220](../../internal/auth/service.go) busca por hash. Columna DB se llama `refresh_token_hash`. Verificado, ya estaba bien.

### 3.2 Goroutine leak SSE como "🔴 crítico"

- **Claim**: "cada desconexión SSE deja ~12 goroutines huérfanas acumulándose".
- **Realidad**: [internal/api/handlers/events.go:67-73](../../internal/api/handlers/events.go) usa `select { case eventCh <- e: default: drop }` — send no-bloqueante. El handler retorna en microsegundos, `done` se cierra, dispatch limpia. No había leak.
- **Lo que sí había**: el bus tenía un `<-done` después del timeout que esperaba indefinidamente — antipattern futuro-frágil. Bajado a 🟡 y arreglado igualmente (commit `35a1d63`).

### 3.3 `useLogout` "usa `resetQueries` en vez de `clear`"

- **Claim**: "caché residual queda vivo".
- **Realidad**: [web/src/api/hooks.ts:896](../../web/src/api/hooks.ts) ya hace `queryClient.clear()`. El auditor leyó mal.

### 3.4 Scanner "no respeta ctx en walk"

- **Claim**: "shutdown bloquea hasta terminar un scan grande".
- **Realidad**: [internal/scanner/scanner.go:289](../../internal/scanner/scanner.go) tiene `if ctx.Err() != nil { return ctx.Err() }` dentro del `WalkDir` callback. La cancelación se honra.

**Lección**: pedir a los auditores LLM que cite líneas reales, no que especulen desde la descripción del proyecto. Verificación manual antes de aplicar fix detectó los cuatro.

## 5. Deuda explícitamente NO abordada

Lista corta de cosas que el audit recomendó pero que para self-hosted single-tenant son low-value o excesivamente invasivas. Cada uno tiene rationale:

- **CSRF binding a sesión** ([internal/api/csrf.go:23](../../internal/api/csrf.go)). Token actual rota cada 24h sin atar a sessionID. Para self-hosted single-tenant la ventana de exfil de un token CSRF es demasiado estrecha para justificar la complejidad (state server-side, rotación coordinada, fallback en logout).
- **Setup endpoint rate-limit en backend**. Nginx ahora lo cubre; el handler sólo se dispara una vez en bootstrap y devuelve 403 silente si ya hay usuarios.
- **bcrypt cost 12 → 13**. Cambio retroactivo encarece el login perceptible. Mejor en futuro como migración explícita con hash-on-login-rotate.
- **gosec medium-severity sweep** (27 findings). Falsos positivos masivos por sanitización vía `pathmap`/allowlist + 6 decisiones de diseño contextuales (G118 goroutines detached, G120 form parsing en upload). Merece su propio PR con `#nosec` razonados, no un drive-by.
- **Refresh TTL 30d → 7d**. Para self-hosted con TVs/dispositivos casuales, 30 días es razonable (Plex tokens duran indefinido). Lo crítico es que estén hasheados — verificado.
- **Tests frontend de hooks críticos** (`useLiveHls`, `useTrickplay`, `useEventStream`). Real follow-up para Fase 5; no entró por scope.

## 7. Convenciones nuevas descubiertas

Añadidas a `conventions.md`:

- **gosec gate**: HIGH/HIGH only en CI; medium se audita manualmente cada cuatrimestre.
- **iframe externos**: lazy-load tras gates de a11y/network, no on-mount.
- **Async cleanup en Promise**: usar `.then(cleanup, cleanup)` en vez de `.finally(cleanup)` cuando la promise puede rechazar y nadie observa el resultado del cleanup-promise (vitest flagea unhandled rejection si no).
